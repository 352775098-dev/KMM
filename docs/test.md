## 1. 测试策略

### 1.1 测试范围

**C++模块测试范围**：
- Controller层：路由管理
- Business层：记忆管理、用户服务、偏好管理、定时任务
- Domain层：数据模型、数据表、知识管理、微服务管理
- Common层：工具类、HTTP助手、JSON解析、雪花ID生成
- Infrastructure层：数据库、向量检索、模型推理、Python服务

**Python模块测试范围**：
- Agent层：情景记忆Agent、语义记忆Agent、核心记忆Agent、事件队列
- Client层：数据库客户端、嵌入客户端、LLM客户端
- Server层：FastAPI服务器、路由处理
- ORM层：数据模型映射
- Tools层：日期转换工具

### 1.2 测试工具

| 测试类型 | 测试工具 | 覆盖率工具 |
|---------|---------|-----------|
| C++单元测试 | GTest | gcov/lcov |
| Python单元测试 | PyTest | pytest-cov |
| API测试 | Postman, curl | - |
| 集成测试 | PyTest, Testcontainers | - |
| 性能测试 | JMeter, Locust | - |
| 安全测试 | OWASP ZAP | - |

### 1.3 测试优先级

| 优先级 | 描述 | 测试覆盖率要求 |
|-------|------|---------------|
| P0 | 核心功能，必须测试 | 100% |
| P1 | 重要功能，重点测试 | ≥80% |
| P2 | 一般功能，常规测试 | ≥60% |
| P3 | 辅助功能，选择性测试 | ≥40% |
